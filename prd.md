📋 [PRD] 홀시(Holsi) - 월경 주기 & 영양제 관리 웹 서비스슬로건: 이제는 혼자 고민하지 마. 아는 만큼 든든해지는, 나를 돌보는 가장 다정한 선택
1. 프로젝트 개요항목내용프로젝트명홀시 (Holsi - '홀몬시스터즈'의 줄임말)핵심 가치Empowerment (내 몸을 스스로 관리하고 지킬 수 있는 데이터와 지식 제공)한 줄 설명여성의 건강한 일상을 위한 '월경 주기 예측' 및 '맞춤형 영양제 알람' 로컬 웹 서비스프로젝트 유형프론트엔드 중심 로컬 웹 서비스 (PWA 기반)예상 난이도초중급
2. 사용자 시나리오구분상세 내용누가 (Who)바쁜 일상 속 자기 관리에 관심이 많고, 호르몬 주기에 맞춰 영양제를 꼼꼼히 챙기고 싶은 2030 여성.언제 (When)매일 영양제 복용 알림이 올 때, 생리 시작/종료 시점, 주기별 컨디션 변화가 궁금할 때.왜 (Why)생리 달력과 영양제 리마인더를 앱 하나로 통합하여 편리하게 관리하고, 내 몸의 데이터를 통해 스스로를 더 잘 돌보기 위해.
3. 핵심 기능 목록
✅ 필수 기능 (MVP)기능명설명내 주기 한눈에 보기"생리 시작/끝" 버튼 기록, 다음 예정일 및 가임기 자동 계산영양제 챙겨주기영양제 이름, 시간, 복용 주기(매일/특정 기간) 등록 및 관리잊지 않게 알려드려요설정 시간에 웹 푸시 알림, 오늘의 복용 완료 체크박스 제공로그인 없이 바로 시작브라우저 Local Storage를 활용한 데이터 저장 (무설치/무가입)
✨ 선택 기능 (고도화)기능명설명지금 내 몸에 필요한 팁주기별 맞춤형 다정한 호르몬 가이드 메시지 (홀몬언니의 조언)오늘의 컨디션 기록이모지와 가벼운 체크박스로 증상 기록내 데이터 간직하기기기 변경 대비용 데이터 백업/복구 (JSON 파일 다운로드/업로드)

5. 화면 구성 (하단 탭 구조)
• 🏠 홈 (오늘의 홀시)
    ◦ 최상단: "🚨 터짐주의 D-3" 위젯 (가장 눈에 띄게 배치)
    ◦ 중단: 주기별 홀몬언니 조언 (예: "배란기가 다가와요! 따뜻한 물을 자주 마셔주세요")
    ◦ 하단: 오늘의 영양제 체크리스트 및 완료 체크박스
• 📅 달력 (내 주기)
    ◦ 상단: 월간 달력 (생리/배란기/예정일을 색상으로 직관적 표시)
    ◦ 하단: 선택한 날짜의 생리 시작/종료 기록 버튼 및 컨디션 기록 영역
• 💊 영양제 (영양제 서랍)
    ◦ 중앙: 등록된 영양제 목록 카드 (복용 시간, 주기 표시)
    ◦ 우측 하단: 플로팅 액션 버튼(+)으로 새 영양제 추가 모달 호출
• ⚙️ 설정
    ◦ 알림 켜기/끄기 토글
    ◦ 데이터 백업 및 복구 버튼
    ◦ 앱 버전 및 정보
6. 상세 기능 명세 (주요 로직)
• 주기 예측 알고리즘:
    ◦ 최근 3~6개월 기록을 바탕으로 평균 주기 및 기간 산출. (기록 부족 시 기본값 28일 적용)
    ◦ 최근 생리 시작일 + 평균 주기 = 다음 생리 예정일
    ◦ 다음 생리 예정일 - 14일 = 배란일
• 알림 구동 방식:
    ◦ Service Worker를 등록하여 브라우저가 닫혀 있어도 정해진 시간에 푸시 이벤트 발생.
    ◦ 알림 문구는 배열 형태로 여러 개 준비하여 랜덤 출력 ("건강 챙겨야지 ~" 등).
• 데이터 스키마 (JSON 객체 구조):
    ◦ settings: 알람 설정 유무, 주기 기본값 등
    ◦ records: 생리 기록 날짜 리스트 Array
    ◦ pills: 등록된 영양제 정보 Array (id, 이름, 시간, 복용 주기 등)
    ◦ intake: 영양제 복용 완료 날짜 및 시간 기록 로그
7. 디자인 가이드
• 메인 컬러 (Primary): 비비드 오렌지 (Vivid Orange)
    ◦ 활력, 에너지, 주도성을 상징. 주요 버튼 및 '터짐주의' 등 강조 요소에 적극 사용.
• 서브 컬러 (Secondary):
    ◦ 배경: 소프트 크림 (Soft Cream) - 눈이 편안하고 따뜻한 느낌.
    ◦ 텍스트: 다크 네이비 (Dark Navy) - 완전한 블랙보다 세련되고 부드러운 가독성 확보.
• 타이포그래피: Pretendard
    ◦ 모바일 환경에서 가독성이 뛰어나며 직관적이고 굵직한 폰트 적용.
• UI/UX 스타일:
    ◦ 둥근 모서리 (Border-radius): 크고 둥글게 처리하여 친근하고 다정한 인상 부여.
    ◦ 카드형 레이아웃: 여백을 충분히 주어 복잡하지 않고 깔끔한 토스(Toss) 스타일 지향.
8. 제약 사항 및 해결 방안
• 데이터 유실 위험 (Local Storage의 한계):
    ◦ 이슈: 사용자가 브라우저 캐시 및 방문 기록을 삭제하면 모든 데이터가 초기화됨.
    ◦ 해결 방안 1 (온보딩 안내): 앱 최초 실행 시 "언니가 알려주는 주의사항! 브라우저 기록을 지우면 데이터가 날아가요 🥲" 안내 팝업 제공.
    ◦ 해결 방안 2 (주기적 백업 넛지): 매월 생리가 끝나는 시점에 "안전하게 파일로 한 번 저장해둘까요?" 라며 [데이터 백업하기] 기능을 자연스럽게 유도.
• iOS 웹 푸시 제약:
    ◦ 이슈: 아이폰(Safari)의 경우 웹사이트를 '홈 화면에 추가'해야만 Web Push API가 정상 작동함.
    ◦ 해결 방안: iOS 기기 접속 시, 화면 상단에 "홀시 알림을 받으려면 하단 공유 버튼을 눌러 '홈 화면에 추가'를 해주세요!" 형태의 안내 배너 상시 노출.
